# date-plugin v1.0 第十五轮计划

## 本轮目标

把仓库从“版本准备和发布预检可执行”继续推进到“GitHub Release 内容可直接导出”，补齐 `release:notes`，让 tag、标题和 release body 都能从当前版本说明自动生成。

## 范围

### 1. Release body 导出脚本

- 新增 `scripts/release-notes-export.mjs`
- 默认按 `package.json` 当前版本读取对应的 `docs/release-notes-v<version>.md`
- 输出 `Tag`、`Title` 和完整 release body

### 2. 文件输出能力

- 支持 `--output <path>` 将 release body 导出到指定文件
- 支持 `--version <version>` 导出非当前版本的 release notes

### 3. 发布文档同步

- 新增 `npm run release:notes`
- 在 README、发布流程和检查清单中说明它的使用方式
- 在 preflight 中纳入对 `release:notes` 脚本入口的校验

## 本轮不做

- 不直接调用 GitHub CLI 创建 release
- 不自动上传构建产物
- 不自动推送 tag

## 风险点

- 导出脚本只负责读取和包装 release notes，不负责保证正文质量
- 如果 release notes 文档结构未来大改，导出格式需要同步调整
- 当前输出更偏“实用发布文本”，不是 GitHub API 专用 JSON 格式

## 验证方式

1. 先执行缺失的 `node scripts/release-notes-export.mjs`
2. 新增脚本后执行 `npm run release:notes`
3. 验证 `--output` 能正常生成文件
4. 执行 `npm run release:preflight`，确认发布链路未破坏

## 预期产出

- 当前版本的 GitHub Release 文本可以一条命令导出
- 发布动作从“手工打开文档复制内容”进一步收敛为脚本化流程
