# date-plugin v1.0 第十七轮计划

## 本轮目标

把仓库从“可预演 GitHub Release 创建”继续推进到“可预演完整发布流程”，补齐 `release:publish`，把 preflight、notes、回归、tag、push 和 release 创建串成一条可 dry-run 的发布总控链路。

## 范围

### 1. 发布总控脚本

- 新增 `scripts/release-publish.mjs`
- 默认只输出完整发布计划，不直接执行外部动作
- 显式传入 `--execute` 时才真正执行发布序列

### 2. 前置风险检查

- 检查当前分支名
- 检查工作区是否干净
- 检查目标 tag 是否已存在
- 检查当前版本 release notes 是否存在

### 3. 发布流程串联

- 串联 `release:preflight`、`release:notes`、`test:ci`
- 串联 `git tag`、`git push` 和 `release:github -- --execute`
- 支持 `--target` 覆盖 GitHub Release 目标分支或提交

## 本轮不做

- 不自动生成构建产物
- 不在 CI 中自动触发正式发布
- 不绕过 dry-run 默认安全边界

## 风险点

- 真正执行发布时依赖本机 `git` 与 `gh` 已登录且远端权限正确
- 工作区当前存在较多未提交改动，dry-run 会明确暴露这一点
- 发布总控脚本只负责串联动作，不替代人工确认 release notes 正文质量

## 验证方式

1. 先执行缺失的 `node scripts/release-publish.mjs`
2. 新增脚本后执行 `npm run release:publish`
3. 确认 dry-run 输出包含分支、tag、clean state 和完整发布计划
4. 执行 `npm run release:preflight`，确认链路未破坏

## 预期产出

- 项目已经具备“从预检到远端 release”的完整脚本化发布骨架
- 真正发布前只需在 dry-run 输出确认无误后再显式执行
