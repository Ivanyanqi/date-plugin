# date-plugin v1.0 第二轮计划

## 本轮目标

让现有 legacy 包装层开始真正接入新的 `src/core`，减少重复实现，为后续键盘交互、约束规则和公共 API 建设打底。

## 范围

### 1. 接入共享 core

- 让 `date.plugin.js` 使用 `src/core/calendar.js`
- 让 `date.plugin.ext.js` 使用 `src/core/calendar.js`
- 让两个 legacy 文件共享 `src/core/format.js`

### 2. 保持兼容行为

- `index.html` 继续正常工作
- `regression.html` 继续通过
- `tests/core/calendar.test.html` 持续通过

### 3. 为后续重构留接口

- 尽量让 legacy 文件中的“纯逻辑”和“DOM 逻辑”分界更明显
- 避免继续在 legacy 文件里新增重复工具函数

## 本轮不做

- 不引入打包工具
- 不引入 npm 依赖
- 不做键盘导航
- 不做 options 体系
- 不做新的 UI 样式升级

## 风险点

- 当前 demo 页面依赖浏览器全局对象，直接引入 ESM 需要谨慎
- 本地 `node` 运行时损坏，CLI 侧测试暂时无法作为主验证方式
- legacy 文件当前结构比较扁平，迁移时要控制改动面，避免把兼容层一次性推翻

## 验证方式

1. 打开 `tests/core/calendar.test.html`，确认纯逻辑测试通过
2. 打开 `regression.html`，确认交互回归通过
3. 打开 `index.html`，手动验证选择、切月、输入回填正常

## 预期产出

- legacy 文件去掉一部分重复纯逻辑
- 新 core 成为日历计算与格式化的唯一来源
- 后续第三轮可以直接进入实例状态对象与键盘交互建设
