# date-plugin 产品路线图

## 愿景

`date-plugin` 将从一个轻量演示项目演进为适用于现代 Web 应用的生产级日期选择器。产品目标是先把“单日期选择”做到稳定、易集成、可访问，再逐步扩展到日期范围、日期时间和更完整的日期输入工具集。

## 产品原则

1. 先保证稳定性，再扩展功能数量
2. 可访问性是核心能力，不是后补项
3. 核心实现保持与框架无关
4. 默认体验清晰，同时保留强配置能力
5. 对接入方保持低心智负担
6. 后续文档、计划、说明默认使用中文

## 产品阶段

### 阶段 0：Demo 稳定化

这一阶段已经开始执行。

- 修复重复事件绑定
- 避免多个浮层面板互相干扰
- 明确当前两种使用方式
- 增加浏览器回归页，覆盖已知交互问题

### 阶段 1：v1.0 单日期选择器

这是第一个正式产品目标。

**用户结果**

- 用户可以在桌面端和移动端稳定选择单个日期
- 接入方可以通过清晰、可预测的 API 集成组件
- 常见的约束、格式化和本地化需求可以满足

**功能范围**

- 单日期选择
- 手动输入与面板选择并存
- `minDate` 与 `maxDate`
- 禁用日期规则
- 基础本地化文案与格式化
- 键盘导航与焦点管理
- 响应式浮层定位
- 基于 CSS 变量的主题能力
- 稳定的实例级公共 API

**不纳入 v1.0**

- 日期范围选择
- 时间选择
- 多月并排布局
- React/Vue 等框架适配层
- 超出 `Today` 和 `Clear` 之外的大量快捷预设

### 阶段 2：v1.5 高级日期选择

- 日期范围选择
- 快捷预设
- 月份与年份快速切换
- 大屏多月并排展示

### 阶段 3：v2.0 日期与时间工具集

- 日期时间选择器
- 时区处理策略
- React 和 Vue 适配器
- 更多可组合的输入原语

## 工作流主线

### UI 主线

- 建立基于 CSS 变量的视觉系统
- 明确输入框、头部、星期栏、日期网格、底部操作区层级
- 定义 hover、selected、disabled、focused、today、outside-month 等状态
- 为小屏设备增加响应式展示策略
- 支持浅色、深色和高对比主题

### 交互主线

- 构建稳定的打开与关闭机制
- 增加完整的键盘导航
- 保证手动输入与面板状态同步
- 确保关闭后焦点能正确回到输入框
- 统一日期约束与无效状态处理

### 集成主线

- 定义一套稳定公共 API
- 同时支持浏览器直引和模块化接入
- 完整记录事件、方法和配置项
- 明确销毁与清理行为

### 工程主线

- 将纯日期逻辑与 DOM 渲染分离
- 为日期计算引入确定性测试
- 保留浏览器回归覆盖核心交互路径
- 在 v1.0 后续阶段引入打包、发布和语义化版本能力

## 里程碑计划

### 里程碑 1：基础层

- 抽离纯日历与格式化逻辑
- 明确内部架构边界
- 保持现有 demo 行为可用
- 补齐项目指导文档和阶段计划

### 里程碑 2：可访问核心

- 实现键盘导航
- 增加焦点进入、保持与返回
- 引入 ARIA 角色与标签
- 增加无效态和禁用态

### 里程碑 3：可配置产品层

- 引入公共 options 对象
- 支持本地化和格式化
- 支持日期约束
- 完成主题变量和面板定位

### 里程碑 4：发布候选版

- 完成浏览器兼容性检查
- 刷新文档和示例
- 引入打包与版本策略
- 补齐 QA 清单和发布说明

## v1.0 完成定义

- 单日期选择器可在现代桌面与移动浏览器中稳定工作
- 支持完整键盘操作
- 屏幕阅读器核心语义可用
- 手动输入与面板选择持续同步
- 日期约束与禁用规则生效
- 公共 API 有稳定文档
- 核心行为具备自动化回归覆盖
- Demo 与文档都以真实支持的 API 为准
