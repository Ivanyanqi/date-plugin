# date-plugin v1.0 第三轮计划

## 本轮目标

在已经具备 `locale`、`format`、`defaultValue`、键盘导航和日期约束的基础上，补齐“生产可用”最关键的一层输入同步能力，让手动输入、面板状态和业务回调开始形成闭环。

## 范围

### 1. 输入值同步

- 支持在打开面板前解析当前 input 值
- 当用户手动输入合法日期时，同步可见月份和选中态
- 当用户输入非法日期时，保留原始输入并暴露错误状态

### 2. 生命周期与业务回调

- 增加 `onOpen`
- 增加 `onClose`
- 增加 `onChange`
- 增加 `onInputError`

### 3. 行为选项补齐

- 增加 `allowManualInput`
- 增加 `closeOnSelect`
- 为后续 `showToday`、`showClear` 保留 options 入口

## 本轮不做

- 不做日期范围选择
- 不做时间选择
- 不做框架适配层
- 不做完整视觉系统升级
- 不引入新的构建工具链

## 风险点

- 当前 legacy 包装层仍承担较多事件编排职责，新增回调时要避免再次把状态源分散
- 手动输入与面板选择并存后，非法输入与合法未选中输入要区分清楚
- 本地 `node` 运行时仍不可用，因此本轮验证仍以浏览器测试页为主

## 验证方式

1. 新增输入同步测试页，覆盖合法输入、非法输入和回调触发
2. 回归 `tests/public/date-picker-api.test.html`
3. 回归 `tests/options/display-options.test.html`
4. 回归 `tests/interaction/keyboard-navigation.test.html`
5. 回归 `regression.html`

## 预期产出

- 手动输入与面板状态不再脱节
- 公共 API 开始具备可接入业务系统的事件能力
- v1.0 后续可以直接进入面板定位、主题变量和底部操作区建设
