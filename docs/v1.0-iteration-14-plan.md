# date-plugin v1.0 第十四轮计划

## 本轮目标

把当前仓库从“版本链路已动态化”继续推进到“版本升级动作半自动化”，补齐 `release:prepare`，让下一个 beta 版本准备时不再手工同步 [package.json](../package.json)、[package-lock.json](../package-lock.json)、[README.md](../README.md) 引用和 release notes 草稿。

## 范围

### 1. 版本准备脚本

- 新增 [scripts/release-prepare.mjs](../scripts/release-prepare.mjs)
- 支持 `node scripts/release-prepare.mjs <version> --dry-run`
- 自动更新 [package.json](../package.json)、[package-lock.json](../package-lock.json)、[README.md](../README.md) 中的 release notes 引用

### 2. 发布文档草稿生成

- 当目标版本的 release notes 不存在时自动创建草稿
- 当 [CHANGELOG.md](../CHANGELOG.md) 中缺少目标版本条目时自动插入新标题
- 保持已存在 release notes 不被重复覆盖

### 3. 开发入口与发布文档同步

- 新增 `npm run release:prepare -- <version>`
- 更新发布流程与检查清单，说明 prepare / preflight / test:ci 的配合关系

## 本轮不做

- 不自动生成 Git tag
- 不自动创建 GitHub Release
- 不自动推送 npm

## 风险点

- 自动生成的 release notes 和 changelog 只能提供草稿骨架，仍需要人工润色
- 如果 README 中未来出现多处版本化 release notes 引用，脚本替换策略需要继续收紧
- 版本准备脚本属于写操作，必须保留 dry-run 预演能力

## 验证方式

1. 先执行缺失的 `node scripts/release-prepare.mjs 1.0.0-beta.3 --dry-run`
2. 新增脚本后执行 dry-run，确认输出计划正确
3. 再执行 `npm run release:preflight`，确认当前版本链路未被破坏

## 预期产出

- 下一个 beta 版本准备动作可通过一条命令完成大部分重复性修改
- 发版流程从“人工同步版本号”继续收敛为“prepare + preflight + test:ci”
