# date-plugin v1.0 实施计划

> **给执行型 agent 的说明：** 推荐使用 `superpowers:subagent-driven-development` 或 `superpowers:executing-plans` 按任务逐项推进。步骤使用复选框 `- [ ]` 便于追踪。

**目标：** 交付一个具备生产级基础能力的单日期选择器底座，包括稳定实例状态、明确技术边界，以及第一轮纯日期核心抽离。

**架构策略：** 新能力优先沉淀到 `src/` 模块中，同时保持当前浏览器 demo 可继续工作。先以纯工具模块作为新权威实现，再逐步让 legacy 包装层与交互逻辑接入这些模块。

**技术栈：** 原生 JavaScript、浏览器 demo、项目内 HTML 回归页；待本地 Node 运行时恢复后，再补 Node 原生测试与正式打包能力。

- [ ] **步骤 1：抽离纯日历模型**

将原本嵌在 [date.plugin.js](../date.plugin.js) 中的日历计算逻辑提取为纯函数，返回结构如下：

```js
{
  year: 2026,
  month: 6,
  dateList: [
    {
      date: 1,
      showDate: 1,
      month: 6
    }
  ]
}
```

- [ ] **步骤 2：抽离格式化工具**

创建补零与标准 `YYYY-MM-DD` 日期格式化工具：

```js
export function addPadding(value) {
  return value < 10 ? "0" + value : String(value);
}

export function formatDate(year, month, date) {
  const normalized = new Date(year, month - 1, date);
  return normalized.getFullYear() + "-" +
    addPadding(normalized.getMonth() + 1) + "-" +
    addPadding(normalized.getDate());
}
```

- [ ] **步骤 3：统一导出 Core API**

通过 [src/core/index.js](../src/core/index.js) 暴露统一内部 API：

```js
export { getMonthAndDate } from "./calendar.js";
export { addPadding, formatDate } from "./format.js";
```
