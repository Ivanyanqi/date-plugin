# date-plugin v1.0 第六轮计划

## 本轮目标

补齐小屏设备下的稳定展示模式，让日期面板在移动端不再只是“缩小后的桌面浮层”，而是具备明确结构的 bottom-sheet 形态。

## 范围

### 1. 移动端 sheet 模式

- 小屏视口下切换为 `data-placement="sheet"`
- 面板固定吸附到底部，左右留出统一安全边距
- 移动模式下暴露 `data-mobile="true"`

### 2. 移动端操作区布局

- Today / Clear 按钮在 sheet 模式下平均分布
- 头部、日期网格和底部操作区的内边距做移动端微调
- 让底部区域更接近可触控尺寸

### 3. 桌面与移动端分流

- 桌面模式继续保留 `top / bottom` 浮层定位
- 移动模式不再依赖输入框左右对齐
- 保持现有键盘导航、输入同步和回调行为不回退

## 本轮不做

- 不做遮罩层和锁滚
- 不做拖拽关闭
- 不做 iOS / Android 平台差异适配
- 不做底部动画和手势系统
- 不做完整移动端视觉主题重设

## 风险点

- 不能让移动端定位分支影响桌面端 top / bottom 自动翻转
- 测试里使用的小屏模拟必须稳定，避免把真实浏览器窗口大小耦合进断言
- sheet 模式下 footer 拉伸不能破坏桌面端按钮宽度

## 验证方式

1. 新增移动端 sheet 测试页
2. 回归 visual states / footer positioning 测试
3. 回归输入同步、ARIA、键盘导航、公共 API
4. 回归 `regression.html`

## 预期产出

- v1.0 在移动端具备更清晰的基础展示模式
- 后续可以继续进入遮罩层、锁滚和触控细节优化
- 面板布局已经具备桌面浮层与移动 sheet 两套稳定基础
