# date-plugin v1.0 第五轮计划

## 本轮目标

继续把面板从“有样式”推进到“可持续扩展”，补齐状态视觉钩子、定位语义和打开后重定位能力，让主题系统、移动端适配和视觉回归都有稳定依赖点。

## 范围

### 1. 状态视觉钩子

- 为 selected 状态补齐稳定类名和 `aria-selected`
- 为 today 状态补齐稳定类名和 `aria-current`
- 为 outside-month 状态补齐稳定类名和 `data-outside`
- 为 disabled 状态补齐稳定类名和 `aria-disabled`

### 2. 定位语义

- 面板打开后暴露 `data-placement="top|bottom"`
- 右侧与底部空间不足时保持可预测定位
- 为后续动画和移动端布局保留 placement 元数据

### 3. 打开后重定位

- 窗口 `resize` 时重算面板位置
- 页面 `scroll` 时重算面板位置
- 销毁实例时清理定位相关监听

## 本轮不做

- 不做视觉快照测试
- 不做动画方向切换
- 不做移动端 bottom-sheet 模式
- 不做月份与年份切换器
- 不做深色主题变量集

## 风险点

- 语义属性增加后，不能破坏现有键盘导航和焦点逻辑
- scroll / resize 重定位要避免在实例销毁后留下全局监听
- selected、today、disabled 等状态可能同时叠加，类名规则要保持清晰

## 验证方式

1. 新增视觉状态与 placement 测试页
2. 回归 Today/Clear 与定位测试
3. 回归输入同步、键盘导航、ARIA 与公共 API
4. 回归 `regression.html`

## 预期产出

- 样式层和语义层的状态表达更稳定
- 后续主题与移动端适配不再依赖脆弱 DOM 推断
- v1.0 下一轮可以继续进入移动端布局模式和更完整的视觉 polish
