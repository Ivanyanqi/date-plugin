# date-plugin v1.0 第九轮计划

## 本轮目标

补齐系统动效偏好适配，让移动端 sheet 在 `prefers-reduced-motion` 场景下自动降级为无延迟开关，进一步提升可访问性和产品完成度。

## 范围

### 1. reduced-motion 检测

- 读取 `prefers-reduced-motion: reduce`
- 为 sheet 和 overlay 暴露 `data-reduced-motion="true|false"`
- 让样式层可以根据该元数据关闭过渡

### 2. 开关动画降级

- reduced-motion 模式下跳过 `opening` 延迟
- reduced-motion 模式下跳过 `closing` 持有阶段
- 保持 overlay、锁滚和 body 恢复逻辑一致

### 3. 现有动画兼容

- 默认模式继续保留 `opening / open / closing / closed`
- 不影响桌面模式、输入同步和键盘交互
- 不影响移动端 safe-area 和 overlay 行为

## 本轮不做

- 不做运行时监听系统偏好变化
- 不做 motion 配置项开放
- 不做手势物理动画
- 不做复杂 easing 或时长分级
- 不做动画快照测试

## 风险点

- reduced-motion 分支不能绕过 overlay / 锁滚释放
- 不能让默认模式的动画状态被错误短路
- 元数据增加后要保持 DOM 和测试断言稳定

## 验证方式

1. 新增 mobile reduced-motion 测试页
2. 回归 mobile animation / overlay / sheet 测试
3. 回归键盘导航、ARIA、公共 API 与输入同步
4. 回归 [regression.html](../regression.html)

## 预期产出

- 组件对系统动效偏好更友好
- v1.0 的移动端交互更接近发布可用标准
- 后续可以转向发布清单、兼容性和文档收尾
