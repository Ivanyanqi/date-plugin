# date-plugin v1.0 第十八轮计划

## 本轮目标

把发布总控脚本从“可展示动作顺序”继续推进到“可展示真实发布阻塞点”，补齐 `origin`、upstream、`gh` 登录状态和本地/远端 tag 风险的诊断输出。

## 范围

### 1. 发布诊断增强

- 在 `release:publish` dry-run 中显示 `origin` 远端地址
- 显示当前分支 upstream 状态
- 显示 GitHub CLI 登录状态
- 保留工作区是否干净、tag 是否已存在等高风险信息

### 2. 执行前阻塞校验

- `--execute` 前拒绝未登录 `gh` 的执行
- `--execute` 前拒绝缺失 `origin` 远端的执行
- 保持默认 dry-run，不自动触发任何远端写操作

### 3. 文档同步

- 在 [README.md](../README.md) 和 [docs/release-process.md](./release-process.md) 中说明 `release:publish` 现在会输出发布环境诊断
- 让最终发布前的判断更依赖脚本证据，而不是人工猜测

## 本轮不做

- 不自动检测远端 tag 是否已存在
- 不自动调用 `gh auth login`
- 不自动修复工作区状态

## 风险点

- `gh auth status` 未登录时会返回非零退出码，脚本必须把它当成状态而不是直接崩溃
- 当前仍以本地诊断为主，远端网络可达性不在默认 dry-run 范围内

## 验证方式

1. 运行 `npm run release:publish`
2. 确认输出包含 remote、upstream、gh auth、clean state
3. 再执行 `npm run release:preflight`，确认链路未破坏

## 预期产出

- 发布总控 dry-run 不只是“要做什么”，还会明确“当前为什么还不能安全发版”
