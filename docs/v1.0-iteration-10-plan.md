# date-plugin v1.0 第十轮计划

## 本轮目标

补齐对业务接入更友好的月份控制 API，让外部可以直接控制当前面板月份，并在翻月时获得稳定的回调信号。

## 范围

### 1. 公共月份控制方法

- 增加 `setMonth(year, month)`
- 支持在面板打开前预设可见月份
- 支持在面板打开后立即切换并重渲染当前月份

### 2. 月份变化回调

- 增加 `onMonthChange`
- 上一月、下一月和 API 设置月份时统一触发
- 回调返回 `source / year / month`

### 3. 接入层一致性

- `DatePicker` 公共类暴露 `setMonth`
- legacy 包装层实例同步具备 `setMonth`
- 不影响已有输入同步、键盘导航、移动端 sheet 和动画逻辑

## 本轮不做

- 不做月份/年份下拉选择器
- 不做多月并排视图
- 不做受控 value / month 双向绑定模型
- 不做 onMonthChange 的节流或批量触发
- 不做历史月份缓存

## 风险点

- setMonth 在打开前和打开后都要保持一致行为
- onMonthChange 不能因为初始化解析或默认值回填而产生噪声回调
- 浏览器模块缓存可能影响新增 API 测试，需要保持测试加载路径稳定

## 验证方式

1. 新增 month control API 测试页
2. 回归 public API 测试
3. 回归输入同步、显示选项、键盘导航和移动端测试
4. 回归 [regression.html](../regression.html)

## 预期产出

- 外部业务可以直接控制当前月份视图
- 面板翻月行为具备更明确的业务事件出口
- v1.0 的公共 API 更接近生产集成可用状态
