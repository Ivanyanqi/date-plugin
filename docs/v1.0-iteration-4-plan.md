# date-plugin v1.0 第四轮计划

## 本轮目标

把日期面板从“功能可用”推进到“产品可用”的基础形态，补齐底部快捷操作、主题变量和基础视口定位，让组件开始具备可发布前的 UI 底座。

## 范围

### 1. 底部操作区

- 增加 `showToday`
- 增加 `showClear`
- Today 操作支持按当前 `format` 回填输入框
- Clear 操作支持清空当前值并触发统一变更回调

### 2. 主题变量

- 将 `date.style.css` 切换为 CSS 变量驱动
- 补齐面板、文字、强调色、禁用态、底部操作区等基础变量
- 为后续浅色/深色/品牌主题改造预留稳定样式接口

### 3. 基础面板定位

- 打开面板时按视口进行水平夹紧
- 在右侧空间不足时避免直接溢出屏幕
- 为后续移动端和垂直方向智能翻转保留结构

## 本轮不做

- 不做完整深色主题
- 不做动画系统
- 不做多月布局
- 不做月份/年份快速切换
- 不做框架级样式封装

## 风险点

- 当前面板仍是 legacy 包装层驱动，定位逻辑要避免和已有打开关闭流程互相干扰
- Today / Clear 引入后，按钮行为要与 `closeOnSelect` 和 `onChange` 保持一致
- 样式变量化不能破坏现有浏览器回归页选择器和结构断言

## 验证方式

1. 新增底部操作区与定位测试页
2. 回归键盘导航与约束测试
3. 回归输入同步与公共 API 测试
4. 回归 `regression.html`
5. 手动打开 `index.html` 观察 demo 行为

## 预期产出

- 组件具备更接近生产产品的基础外观和操作区
- 面板在常见窄空间场景下不会直接溢出视口
- v1.0 下一轮可以继续进入移动端适配、Today/Clear 细化和视觉状态完善
