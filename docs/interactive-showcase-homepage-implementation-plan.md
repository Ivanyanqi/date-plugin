# date-plugin Interactive Showcase 首页实现计划

## 目标

把 [index.html](../index.html) 从简单 demo 升级为 interactive showcase 首页，在不改动日期选择器核心逻辑的前提下，提供真实可玩的首页体验。

## 范围

### 1. 页面结构重写

- 重写 [index.html](../index.html)
- 增加 Hero、能力区、接入区、文档入口区
- 增加锚点滚动和页面内导航

### 2. Hero 试玩区

- 使用真实输入框和现有 `datepickerinit.init(...)`
- 提供 locale、format、行为开关等配置切换
- 在切换时重建实例，保证结果真实
- 增加当前值和配置状态回显

### 3. 首页样式增强

- 在 [date.style.css](../date.style.css) 中追加首页专属样式区块
- 建立页面级变量、栅格、卡片、标签、代码块和舞台容器样式
- 确保桌面与移动端可读性

### 4. 内容与文档入口

- 整理能力说明卡片文案
- 增加最小接入代码示例
- 增加发布和文档入口提示

## 本轮不做

- 不重构日期选择器内部实现
- 不增加新的组件核心 API
- 不新增额外业务功能，如范围选择、时间选择

## 风险点

- 首页脚本如果直接堆叠在页面里，容易把展示逻辑写乱，需要控制结构清晰
- 配置切换如果不销毁旧实例，可能引入重复浮层或事件残留
- 首页样式需要和日期选择器面板样式共存，避免视觉污染或选择器冲突

## 实施顺序

1. 完成首页结构设计落地
2. 先实现 Hero 与试玩区
3. 再补能力区和接入区
4. 最后做响应式与样式精修
5. 跑浏览器回归确认无副作用

## 验证方式

1. 打开 [index.html](../index.html)，确认首页结构和视觉明显升级
2. 在试玩区切换配置，确认日期选择器行为跟随变化
3. 回归 [regression.html](../regression.html)
4. 执行 `npm run test:ci`
